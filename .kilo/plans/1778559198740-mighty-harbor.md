# Plano para Corrigir Erro de Template Malformado em Detalhes.vue

## Objetivo
Corrigir o erro "Element is missing end tag" para `<TabsContent value="itens">` causado por estrutura de template aninhada incorreta na seção de itens do pedido.

## Contexto Atual
O erro ocorre na linha 1125 onde `<TabsContent value="itens">` está sendo reportado como faltando tag de fechamento, embora haja `</TabsContent>` na linha 1298. A análise revela que há tags de fechamento extras e faltando na estrutura da lista de itens, incluindo tags `</template>`, `</RecycleScroller>`, `</CardContent>` e `</Card>` incorretas, além de tags `<div>` de fechamento ausentes.

## Estrutura Atual Problemática
```vue
<CardContent class="p-6">
  <div>
    <div v-if="filteredItens.length === 0">...</div>
    <div v-else class="space-y-4">
      <Card v-for="item in filteredItens" ...>
        ...conteúdo do item...
      </Card>
      <!-- Faltam </div> </div> aqui -->
    </template>  <!-- Tag incorreta -->
    </RecycleScroller>  <!-- Tag sem abertura -->
  </CardContent>  <!-- Tag extra -->
</CardContent>  <!-- Tag correta para p-6 -->
```

## Estrutura Correta Necessária
```vue
<CardContent class="p-6">
  <div>
    <div v-if="filteredItens.length === 0">...</div>
    <div v-else class="space-y-4">
      <Card v-for="item in filteredItens" ...>
        ...conteúdo do item...
      </Card>
    </div>
  </div>
</CardContent>
```

## Plano de Implementação

### 1. Remover Tags Incorretas
- Remover `</template>` (linha ~1294)
- Remover `</RecycleScroller>` (linha ~1295)
- Remover `</CardContent>` extra (linha ~1296)

### 2. Adicionar Tags de Fechamento Ausentes
- Adicionar `</div>` para fechar `v-else` (linha ~1294)
- Adicionar `</div>` para fechar o `<div>` externo (linha ~1295)

### 3. Verificar Estrutura Completa
- Confirmar que todas as tags estão corretamente aninhadas
- Validar que o compilador Vue não reporta erros

## Critérios de Pronto
- O arquivo compila sem erros de template
- As abas funcionam corretamente (pedido, cliente, itens)
- Não há tags HTML/Vue malformadas
- A lista de itens é renderizada corretamente

## Riscos e Mitigações
- **Risco**: Quebrar outras partes do template
  - **Mitigação**: Revisar cuidadosamente a estrutura antes e depois das mudanças
- **Risco**: Introduzir novos erros de sintaxe
  - **Mitigação**: Usar ferramentas de linting e compilação para validar</content>
