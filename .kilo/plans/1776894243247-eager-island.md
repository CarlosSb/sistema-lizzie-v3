# Plano: Corrigir Geração de PDF no Sistema Lizzie

## Objetivo
Corrigir o erro de geração de PDF causado pela biblioteca html2pdf incompatível, substituindo por uma alternativa funcional para garantir que a impressão de pedidos funcione corretamente.

## Análise do Problema
- Erro: "html2pdf is not a function" indica problema de import da biblioteca
- Biblioteca atual: "html2pdf": "^0.0.11" (versão antiga e possivelmente quebrada)
- Uso: Geração de PDF para impressão de pedidos (completo/resumido)
- Impacto: Funcionalidade de impressão não funciona, impedindo conclusão de pedidos

## Alternativas Avaliadas
1. **html2pdf.js** (recomendado): Biblioteca oficial mais recente
   - Prós: Mesma API, fácil migração
   - Contras: Pode ter mudanças na API

2. **jsPDF + html2canvas**: Combinação manual
   - Prós: Controle total, bibliotecas maduras
   - Contras: Código mais complexo

3. **Puppeteer**: Geração server-side
   - Prós: Alta qualidade
   - Contras: Requer backend, não client-side

## Plano de Implementação

### Fase 1: Substituir Biblioteca
1. Atualizar package.json: trocar "html2pdf" por "html2pdf.js": "^0.10.1"
2. Atualizar import em PedidoPrint.vue para usar 'html2pdf.js'
3. Ajustar código de geração se necessário (API pode ter mudado)
4. Testar geração de PDF

### Fase 2: Fallback se html2pdf.js falhar
1. Implementar alternativa com jsPDF + html2canvas
2. Criar função wrapper para manter interface consistente
3. Adicionar try/catch para fallback automático

### Fase 3: Testes e Validação
1. Testar impressão completo e resumido
2. Verificar se status é atualizado corretamente após impressão
3. Testar em diferentes browsers

## Riscos
- API da nova biblioteca pode ser diferente
- Possível incompatibilidade com Vue 3
- Dependência de html2canvas pode causar issues

- PDF gerado sem erros
- Impressão funciona em Chrome/Firefox
- Status do pedido atualizado após impressão completo</content>
