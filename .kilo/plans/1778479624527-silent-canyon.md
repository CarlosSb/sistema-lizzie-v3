# Plano para Análise e Correção da Página de Preview do PDF e Funcionamento da Impressão

## Objetivo
Analisar a funcionalidade de preview do PDF e o processo de impressão no sistema Lizzie, identificando e corrigindo bugs que impedem o funcionamento adequado da geração e impressão de PDFs de pedidos.

## Contexto do Problema
- Usuário reportou necessidade de analisar a página de preview do PDF e verificar o funcionamento da impressão
- Durante a análise inicial, foram identificados vários erros de TypeScript e funções ausentes
- A funcionalidade de impressão inclui dois fluxos: preview modal (PedidoPrint) e modal de PDF gerado

## Estado Atual Identificado

### Componente PedidoPrint
- Funciona em modo preview (modal) e modo direto
- No modo preview, exibe o conteúdo do PDF em HTML e permite imprimir via botão "Imprimir"
- A impressão usa html2pdf.js com fallback para jsPDF + html2canvas
- Método printPdfBlob tenta imprimir via iframe, com fallback para nova janela

### Erros Identificados em Detalhes.vue
1. **Erro de TypeScript na linha 404**: Variável local `pdfBlob` sombreia o ref `pdfBlob`, causando tentativa de acesso a `.value` em Blob
2. **Função `showPdfPreview` não definida**: Botão "Ver PDF" chama função inexistente
3. **Função `onPdfGenerated` não definida**: Emit do PedidoPrint não tratado

## Plano de Implementação

### 1. Corrigir Erro de TypeScript em autoPrintCompletedOrder
- Renomear variável local `pdfBlob` para `blob` em autoPrintCompletedOrder
- Ajustar atribuições para usar `pdfBlob.value = blob` e `pdfBlobUrl.value = URL.createObjectURL(blob)`

### 2. Implementar Função showPdfPreview
- Criar função que gera PDF do pedido atual e abre modal de preview do PDF gerado
- Usar endpoint `/api/pdf/pedido/${id_pedido}` com parâmetros apropriados
- Definir função assíncrona com tratamento de erros

### 3. Implementar Função onPdfGenerated
- Criar função que recebe o modo ('complete' | 'summary') como parâmetro
- Fechar modal de preview após geração bem-sucedida
- Opcionalmente, definir printMode para null

### 4. Verificar Funcionalidade de Impressão
- Testar fluxo completo: seleção de modo → preview → geração de PDF → impressão
- Verificar se o PDF gerado contém todas as informações corretas
- Validar fallbacks de impressão (iframe vs nova janela)

### 5. Melhorar Tratamento de Erros
- Adicionar logs apropriados para debugging
- Melhorar mensagens de erro para usuário
- Garantir que estados de loading sejam gerenciados corretamente

## Critérios de Pronto
- Erros de TypeScript resolvidos
- Funções `showPdfPreview` e `onPdfGenerated` implementadas
- Preview do PDF funciona corretamente
- Processo de impressão gera PDF válido e abre diálogo de impressão
- Modal de preview fecha após geração de PDF
- Não há erros no console durante o fluxo

## Sequência Recomendada
1. Corrigir erro de TypeScript em autoPrintCompletedOrder
2. Implementar função showPdfPreview
3. Implementar função onPdfGenerated
4. Testar funcionalidade completa
5. Executar build para verificar ausência de erros

## Considerações Técnicas
- Usar server-side PDF generation quando disponível (`useServerSide` prop)
- Manter compatibilidade com html2pdf.js e jsPDF
- Garantir que estilos CSS sejam aplicados corretamente na geração do PDF
- Considerar performance para pedidos com muitos itens
