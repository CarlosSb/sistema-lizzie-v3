# Plano para Corrigir Erro de Template Inválido em Detalhes.vue

## Objetivo
Corrigir o erro de "Invalid end tag" no arquivo Detalhes.vue causado por estrutura de template malformada nos componentes de abas (Tabs).

## Contexto Atual
O erro ocorre em `</CardContent>` na linha 1108, indicando que há uma tag de fechamento sem correspondência. Análise do código revela que a seção do cliente não está devidamente envolvida em `<TabsContent value="cliente">`, e o fechamento `</Tabs>` está posicionado incorretamente após a aba do pedido, deixando o conteúdo do cliente fora da estrutura das abas.

## Estrutura Atual Problemática
```vue
<Tabs>
  <TabsList>...</TabsList>
  <TabsContent value="pedido">
    <Card>...</Card>
  </TabsContent>
</Tabs>  <!-- Fecha Tabs aqui incorretamente -->
<div>...</div>  <!-- Conteúdo do cliente solto -->
<div>...</div>
<CardContent>...</CardContent>  <!-- CardContent sem abertura -->
```

## Estrutura Correta Necessária
```vue
<Tabs>
  <TabsList>...</TabsList>
  <TabsContent value="pedido">
    <Card>...</Card>
  </TabsContent>
  <TabsContent value="cliente">
    <Card>...</Card>
  </TabsContent>
  <TabsContent value="itens">
    <Card>...</Card>
  </TabsContent>
</Tabs>
```

## Plano de Implementação

### 1. Adicionar `<TabsContent value="cliente">`
- Inserir `<TabsContent value="cliente">` antes da `<Card>` do cliente
- Inserir `</TabsContent>` após o fechamento da `<Card>` do cliente

### 2. Reposicionar o fechamento `</Tabs>`
- Remover o `</Tabs>` incorreto após a aba do pedido
- Garantir que `</Tabs>` feche após a aba dos itens

### 3. Verificar Estrutura Completa
- Confirmar que todas as abas estão corretamente aninhadas
- Validar que não há tags soltas fora da estrutura das abas

## Critérios de Pronto
- O arquivo compila sem erros de template
- As abas funcionam corretamente (pedido, cliente, itens)
- Não há tags HTML malformadas

## Riscos e Mitigações
- **Risco**: Quebrar outras partes do template
  - **Mitigação**: Revisar cuidadosamente a estrutura antes e depois das mudanças

- **Risco**: Introduzir novos erros de sintaxe
  - **Mitigação**: Usar ferramentas de linting e compilação para validar</content>
