# Plano para Ajustar Layout dos PDFs de Pedido

## Objetivo
Atualizar o layout dos PDFs de pedido para usar um design mais moderno e estruturado, baseado no layout HTML fornecido, mantendo a funcionalidade existente.

## Análise Atual vs Desejado

### Layout Atual (PedidoTemplate.php)
- CSS inline otimizado para PDF (fontes pequenas, layout compacto)
- Estrutura baseada em tabelas para compatibilidade com PDF
- Estilos específicos para impressão (font-size pequeno, margens mínimas)
- Layout responsivo limitado ao tamanho de página PDF

### Layout Desejado
- CSS moderno com grid layouts e flexbox
- Design web-like com cores suaves e bordas arredondadas
- Estrutura HTML semântica (header, sections, footer)
- Espaçamento generoso e tipografia melhorada
- Layout responsivo para visualização web/PDF

## Plano de Implementação

### 1. Atualizar CSS no PedidoTemplate.php
- Usar diretamente o CSS do exemplo fornecido, que já está alinhado com as expectativas
- Adaptar os estilos para formato inline no PHP (converter para string)
- Manter compatibilidade com geração PDF (usar estilos inline)
- Preservar classes CSS necessárias para PDF (como page-break-inside)

### 2. Reestruturar HTML do Template
- Atualizar estrutura para usar grid layouts ao invés de tabelas
- Implementar header com flexbox (brand + order-info)
- Converter seções para usar classes .section, .box, .client-grid
- Ajustar tabela de itens para manter compatibilidade PDF
- Atualizar assinaturas para usar grid de 2 colunas

### 3. Ajustar Responsividade e Dimensionamento
- Atualizar tamanhos de fonte para melhor legibilidade
- Ajustar larguras de colunas e espaçamentos
- Garantir que o layout caiba em página A4 (794px width)
- Manter elementos de quebra de página para PDFs longos

### 4. Preservar Funcionalidade Existente
- Manter todas as funções de data binding (clientName, safeText, etc.)
- Preservar lógica condicional (mode complete vs summary)
- Manter sanitização de HTML (função h())
- Garantir que QR code e outras funcionalidades continuem funcionando

### 5. Testar Geração de PDF
- Verificar se o novo layout renderiza corretamente em PDF
- Testar diferentes tamanhos de pedido (com/sem cliente, com/sem observações)
- Validar quebra de página e formatação de tabelas
- Confirmar que o layout funciona em diferentes visualizadores de PDF

## Arquivos a Modificar
- `api/app/Services/PedidoTemplate.php` - Principal arquivo do template
- `api/app/Services/DocumentService.php` - Possivelmente ajustes na preparação de dados se necessário

## Considerações Técnicas
- **Compatibilidade PDF**: Manter estilos inline e estrutura compatível com DomPDF/TCPDF
- **Performance**: Evitar estilos excessivamente complexos que possam afetar renderização
- **Manutenibilidade**: Estruturar CSS de forma organizada e comentada
- **Fallbacks**: Garantir que dados ausentes sejam tratados graciosamente

## Critérios de Pronto
- Layout visualmente idêntico ao HTML fornecido
- PDF gerado corretamente sem quebras de layout
- Todas as funcionalidades existentes preservadas
- Código limpo e bem estruturado
- Testes manuais confirmando funcionamento
