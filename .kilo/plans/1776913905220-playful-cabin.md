## Implement Global Search in Header

### Summary
This plan implements a functional global search feature in the application header that allows users to search across customers, orders, products, and other entities. The search will be debounced, display results in a dropdown, and provide intuitive navigation.

### Current State
The header currently contains a non-functional search input:
```html
<div class="hidden lg:flex items-center bg-muted/50 rounded-lg px-3 w-80 group focus-within:bg-background focus-within:ring-1 focus-within:ring-primary transition-all">
  <Search class="w-4 h-4 text-muted-foreground group-focus-within:text-primary transition-colors" />
  <Input 
    type="text" 
    placeholder="Pesquisar..." 
    class="border-0 focus-visible:ring-0 text-sm bg-transparent h-10 w-full" 
  />
</div>
```

### Implementation Plan

#### 1. Add Reactive State
In `frontend/src/layouts/AppLayout.vue` script setup:
- `searchTerm`: ref('') - bound to input value
- `searchResults`: ref([]) - combined results from all entities
- `isSearching`: ref(false) - loading state
- `searchDebounceTimer`: ref(null) - for debouncing

#### 2. Implement Search Functions
Create individual search functions for each entity:
- `searchClients(term)` - calls `/api/clientes?search=${term}`
- `searchOrders(term)` - calls `/api/pedidos?search=${term}`
- `searchProducts(term)` - calls `/api/produtos?search=${term}`
- `searchUsers(term)` - calls `/api/users?search=${term}` (if applicable)

Each function should:
- Return formatted results with consistent structure
- Include entity type for display purposes
- Handle errors gracefully

#### 3. Implement Debounced Search
- Create `performSearch()` function that:
  1. Clears previous debounce timer
  2. Sets `isSearching = true`
  3. If searchTerm is empty, clear results and return
  4. Otherwise, call all search functions in parallel using Promise.all()
  5. Combine and format results
  6. Update `searchResults` and set `isSearching = false`
- Create `handleSearchInput()` function that sets up debounce (300ms delay)

#### 4. Add Search Results Dropdown
In template, below the search input:
```html
<div v-if="isSearching" class="absolute top-full left-0 right-0 mt-2">
  <!-- Loading indicator -->
  <div class="px-4 py-2 text-center text-sm text-muted-foreground">
    Searching...
  </div>
</div>
<div v-else-if="searchResults.length > 0" class="absolute top-full left-0 right-0 mt-2 w-80 bg-card border rounded-lg shadow-lg z-50 max-h-96 overflow-y-auto">
  <div class="px-4 py-2 border-b">
    <span class="font-semibold text-sm">{{ searchResults.length }} results</span>
  </div>
  <div class="divide-y">
    <div v-for="(result, index) in searchResults" :key="index" class="px-4 py-3 hover:bg-accent/50 transition-colors cursor-pointer" @click="navigateToResult(result)">
      <div class="flex items-start gap-3">
        <!-- Entity icon based on type -->
        <div class="flex-shrink-0 h-8 w-8 flex items-center justify-center bg-primary/10 text-primary rounded-lg">
          <!-- Dynamic icon based on result.type -->
        </div>
        <div class="flex-1">
          <p class="text-sm font-medium">{{ result.title }}</p>
          <p class="text-xs text-muted-foreground">{{ result.subtitle }}</p>
        </div>
      </div>
    </div>
  </div>
  <div v-if="searchResults.length === 0 && searchTerm" class="px-4 py-4 text-center text-sm text-muted-foreground">
    No results found for "{{ searchTerm }}"
  </div>
</div>
```

#### 5. Add Navigation Function
Create `navigateToResult(result)` function that:
- Determines the appropriate route based on result.type
- Navigates to the entity detail page with the result ID
- Closes search dropdown (optional)

#### 6. Enhance Input Binding
Modify the search input to:
- Bind value to `searchTerm`
- Add `@input="handleSearchInput"` event
- Add `@keyup.enter` to trigger immediate search
- Add placeholder text like "Search customers, orders, products..."

#### 7. Styling Considerations
- Ensure dropdown appears below search input
- Add subtle animations for appearance/disappearance
- Maintain consistency with existing UI components
- Handle edge cases (very long results, special characters in search)
