# Plano para Correção do Erro CORS na Geração de PDFs

## Objetivo
Corrigir o erro de Cross-Origin Resource Sharing (CORS) que está bloqueando as requisições para geração de PDFs no endpoint `/api/pdf/pedido/{id}`.

## Contexto do Problema
- Usuário reportou erro CORS ao tentar gerar PDFs
- O cabeçalho `Access-Control-Allow-Origin` está ausente na resposta do servidor
- A requisição é bloqueada pelo navegador, impedindo a geração de PDFs

## Análise Técnica

### Erro Identificado
```
A cross-origin resource sharing (CORS) request was blocked because of invalid or missing response headers of the request or the associated preflight request.
Access-Control-Allow-Origin: Header missing
```

### Causas Possíveis
1. **Servidor não configurado para CORS**: O backend não está enviando os cabeçalhos CORS necessários
2. **Middleware CORS ausente**: Não há middleware configurado para lidar com CORS
3. **Configuração específica por rota**: O endpoint PDF pode precisar de configuração específica
4. **Problema de preflight**: Requisições OPTIONS não estão sendo tratadas corretamente

## Plano de Implementação

### 1. Diagnosticar Problema CORS Atual
- ✅ Middleware CORS existe em `api/app/Http/Middleware/CorsMiddleware.php`
- ✅ Middleware está registrado globalmente em `api/bootstrap/app.php`
- ✅ Rota PDF está protegida por middleware CORS em `api/routes/web.php`
- ❌ Verificar se origem do frontend (porta) está na lista de origens permitidas

### 2. Corrigir Configuração CORS
- Atualizar lista de origens permitidas no CorsMiddleware
- Adicionar mais portas comuns de desenvolvimento (3000, 5173, 8080)
- Garantir que headers sejam aplicados corretamente para respostas binárias (PDF)
- Verificar se middleware está funcionando para requisições OPTIONS

### 3. Verificar Configuração do Frontend
- Confirmar que axios está fazendo requisições diretas para localhost:8000
- Verificar se problema é específico da porta ou configuração
- Testar se mudança para usar proxy do Vite resolve o problema

### 4. Testar Correção
- Reiniciar servidor backend após mudanças
- Testar geração de PDF no frontend
- Verificar se erro CORS foi resolvido
- Validar funcionamento de outros endpoints

### 5. Melhorar Tratamento de Erros (Opcional)
- Adicionar logs de CORS no middleware para debugging
- Implementar melhor tratamento de erros no frontend

## Considerações Técnicas
- **Backend**: Lumen (Laravel micro-framework) em `/api`
- **Frontend**: Vue.js com Axios fazendo requisições diretas para localhost:8000
- **Tipo de Resposta**: `responseType: 'blob'` para PDFs binários
- **Método**: POST com payload JSON
- **Proxy**: Vite proxy configurado mas não utilizado devido a URL absoluta

## Cabeçalhos CORS Atuais
```php
Access-Control-Allow-Origin: http://localhost:5173 (ou origem permitida)
Access-Control-Allow-Methods: POST, GET, OPTIONS, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Accept, Authorization, X-Requested-With, X-API-KEY
Access-Control-Max-Age: 86400
Access-Control-Allow-Credentials: true
```
