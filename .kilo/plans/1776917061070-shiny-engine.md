# Plan: Fix Global Search Implementation in AppLayout.vue

## Problem Analysis

The current global search implementation in `frontend/src/layouts/AppLayout.vue` uses native `fetch()` instead of the authenticated `apiClient`, causing API requests to fail due to missing JWT token. Additionally, the `apiClient` returns an Axios response with a different data structure.

### Root Causes Identified

1. **Authentication Missing**: Search calls use `fetch()` which doesn't include the JWT Authorization header
   - All other pages use `apiClient` which has request interceptor to add tokens
   - Backend returns 401 Unauthorized

2. **Incorrect Response Parsing**: `fetch()` uses `response.json()` while `apiClient` uses `response.data`
   - Current code expects `data.data` structure which is correct for backend
   - But fetch returns plain response, not axios response object

3. **VendedorController Search Not Implemented**: Vendor search endpoint lacks search filtering (already fixed in codebase)

4. **Missing `apiClient` Import**: AppLayout.vue doesn't import the configured axios instance

5. **Debounce Timer Type**: Using `ref(null)` with `setTimeout` returns number in browser, needs proper typing

## Solution Approach

Replace all `fetch()` calls in search functions with `apiClient.get()` using the `params` option for proper query string encoding. This ensures:
- JWT authentication is automatically included
- Request/response handling consistent with rest of codebase
- Automatic token refresh on 401 errors
- Proper error handling through axios interceptors

## Implementation Steps

### 1. Import apiClient
Add: `import apiClient from '@/lib/axios'`

### 2. Update Search Functions (4 functions)
Replace:
```javascript
const response = await fetch(`/api/clientes?search=${encodeURIComponent(term)}`)
if (response.ok) {
  const data = await response.json()
  return data.data?.map(...) || []
}
```

With:
```javascript
const response = await apiClient.get('/api/clientes', {
  params: { search: term }
})
return response.data?.data?.map(...) || []
```

Apply to:
- `searchClients()` → endpoint `/api/clientes`
- `searchOrders()` → endpoint `/api/pedidos`
- `searchProducts()` → endpoint `/api/produtos`
- `searchVendors()` → endpoint `/api/vendedores`

### 3. Fix Response Data Mapping
- Cliente: `id_cliente`, `nome`/`razao_social`, `email`/`telefone`
- Pedido: `id_pedido`, `cliente.razao_social` or `data_pedido`
- Produto: `id_produto`, `produto`, `valor_unt_norde`
- Vendedor: `id_vendedor`, `nome_vendedor`, `email`/`telefone`

### 4. Ensure Debounce Timer Typing
Already handled with `ref<number | null>(null)`.

### 5. Add Result Limit
Limit search results to 5 items per entity to improve dropdown performance:
```javascript
params: { search: term, per_page: 5 }
```

### 6. Error Handling
Keep existing try-catch but leverage axios error responses. No changes needed.

### 7. Cleanup on Unmount
Already added `onUnmounted` to clear debounce timer.

## Expected API Response Structure

All endpoints return:
```json
{
  "success": true,
  "data": [...],  // array of records
  "pagination": { ... }
}
```

The code accesses `response.data.data` where:
- `.data` = axios response property
- `.data` = Laravel response body field
- So double `.data.data` is correct pattern
