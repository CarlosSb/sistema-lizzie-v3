# Plano: Adicionar Fluxo de Conclusão e Documento Imprimível aos Pedidos

## Objetivo
Manter o design intuitivo e funcional do formulário atual de pedidos, sem mudanças significativas na estrutura. Apenas promover a adição de funcionalidades essenciais: confirmação pré-conclusão e documento imprimível em duas versões (completa para cliente e resumida para uso interno), inspirado no workflow legado mas adaptado à simplicidade atual.

## Análise Atual
- **Form.vue**: Formulário único e intuitivo, funcionando bem sem necessidade de mudanças estruturais.
- **Fluxo Atual**: Preenchimento direto → Salvar → Redirecionar para Detalhes.
- **Falta**: Confirmação pré-salvamento e documento imprimível pós-conclusão.

## Proposta de Ajustes (Mínimas Mudanças)
### 1. Preservar Design Atual
- Manter formulário único, layout e funcionalidade existentes.
- Não adicionar etapas ou complexidade desnecessária.

### 2. Adicionar Confirmação Pré-Conclusão
- Modal simples de confirmação aparecendo ao clicar "CONCLUIR PEDIDO", mostrando resumo rápido (cliente, total, itens).
- Sem mudanças no layout do formulário.

### 3. Adicionar Documento Imprimível
- **Integração**: Botões de impressão diretamente na página Detalhes.vue, após conclusão.
- **Versões**:
  - **Completa**: Documento detalhado com todos os dados (cliente, pedido, itens, condições).
  - **Resumida**: Versão simplificada para uso interno.
- **Tecnologia**: Usar `html2pdf` para gerar PDF a partir de HTML, com estilos otimizados para impressão.
- **Implementação**: Componente separado para layout de impressão, acionado pelos botões.

### 4. Melhorias Mínimas
- Garantir validação básica (cliente selecionado, itens adicionados).
- Feedback visual durante salvamento (já parcialmente implementado).

## Etapas de Implementação (Invasivas Mínimas)
1. **Confirmação**: Adicionar Dialog de confirmação em Form.vue, integrado ao botão existente.
2. **Componente de Impressão**: Criar `PedidoPrint.vue` com layouts condicionais para versões completa/resumida.
3. **Integração**: Adicionar botões em Detalhes.vue para acionar impressão/PDF.
4. **Biblioteca**: Instalar `html2pdf` via npm.
5. **Testes**: Validar geração de PDF e impressão.

## Perguntas para Usuário
- Quais condições/termos específicos devem constar no documento completo?
- Prefere PDF gerado ou impressão direta do navegador?
- O documento precisa de assinatura digital?

## Riscos
- Geração de PDF pode variar entre navegadores – testes necessários.
- Adição de confirmação é simples e não impacta usabilidade.
