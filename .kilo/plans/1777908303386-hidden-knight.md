# 📋 **Planejamento: Implementação de Sistema de Geração de PDF com PHP e Vue**

## 🎯 **Objetivo**
Implementar um sistema robusto de geração e visualização de PDFs no Sistema Lizzie v3, migrando da geração client-side atual (html2pdf.js/jspdf) para server-side (TCPDF/Dompdf), com foco especial na impressão de pedidos ao concluir/fechar vendas e outros pontos críticos do sistema.

## 🔍 **Análise do Sistema Atual**

### **Estado Atual da Geração de PDF**
- **Bibliotecas Frontend**: `html2pdf.js`, `jspdf`, `html2canvas` (client-side)
- **Componente Principal**: `PedidoPrint.vue` - Geração client-side com fallback
- **Pontos de Impressão**: Detalhes do pedido (modo completo/resumo)
- **Limitações**: Performance, compatibilidade, sobrecarga no cliente

### **Pontos Críticos de Impressão Identificados**
1. **Fechamento de Pedido**: Impressão automática ao concluir venda (status 4)
2. **Etiquetas de Pedido**: Já existe rota `/pedidos/{id}/etiqueta` (expandir)
3. **Relatórios**: Vendas, vendedores, produtos (Dashboard) - converter para PDF
4. **Carnês de Pagamento**: Para pedidos parcelados (itens_pedidos)
5. **Recibos**: Confirmação de recebimento e entrega

### **Infraestrutura Existente**
- **Backend**: Lumen com controllers de Pedido, Cliente, Produto, Dashboard
- **Frontend**: Vue 3 + TypeScript + componentes UI (Reka UI)
- **Banco**: MySQL com relacionamentos completos (pedidos → clientes, vendedores, itens)
- **Status de Pedido**: 1=Aberto, 2=Pendente, 3=Cancelado, 4=Concluído

## 🛠️ **Arquitetura Proposta**

### **Tecnologias Atuais vs. Propostas no Plano**

| Aspecto | Plano Original (React/Next.js) | Equivalente PHP + Vue |
|---------|--------------------------------|----------------------|
| **Geração de PDF** | @react-pdf/renderer (server-side) | TCPDF, Dompdf, ou wkhtmltopdf via PHP |
| **Visualização** | react-pdf (client-side) | PDF.js ou vue-pdf no Vue |
| **Templates** | Componentes React | Templates Blade/PHP ou HTML + CSS |
| **API** | Next.js API Routes | Lumen Controllers/Routes |
| **Frontend** | React Components | Vue Components |
| **Tipagem** | TypeScript | PHPStan ou sem tipagem forte |

### **Pontos de Viabilidade**

#### ✅ **Altamente Viável**
- **Geração Server-side**: PHP tem bibliotecas maduras para PDF (TCPDF usado há décadas)
- **Integração Backend**: Lumen facilita criação de APIs REST para geração
- **Visualização Client-side**: Vue pode integrar PDF.js facilmente
- **Templates Dinâmicos**: PHP pode renderizar HTML/CSS como templates
- **Performance**: Geração server-side evita sobrecarga no cliente

#### ⚠️ **Desafios Técnicos**
- **Templates Baseados em Componentes**: Vue não tem equivalente direto ao @react-pdf/renderer
- **Estilização**: CSS-in-JS vs. CSS tradicional pode requerer adaptação
- **Type Safety**: Menor rigor em PHP comparado ao TypeScript
- **Curva de Aprendizado**: Desenvolvedores precisam aprender bibliotecas PDF do PHP

## 🛠️ **Arquitetura Proposta**

### **Estrutura de Dados (PHP)**
```php
class PrintData {
    public Pedido $pedido;
    public Cliente $cliente;
    public ?Vendedor $vendedor;
    public array $itens = []; // ItemPedido[]
    public array $pagamentos = []; // Para carnês
    public ?string $qrCodeUrl;
    public DateTime $generatedAt;
    public ?string $printedBy;
    public array $calculo = []; // Dados de cálculo do pedido
}

class PrintOptions {
    public string $templateId; // 'pedido', 'etiqueta', 'carne', 'recibo'
    public string $format = 'pdf';
    public bool $includeQR = false;
    public string $paperSize = 'a4'; // 'a4', 'a5', 'letter'
    public string $orientation = 'portrait'; // 'portrait', 'landscape'
    public int $copies = 1;
}
```

### **Fluxo de Geração**
```
Usuário solicita PDF → Vue chama API → Lumen busca dados → 
PdfService renderiza template → TCPDF/Dompdf gera buffer → 
Retorna blob/base64 → Download/Preview
```
