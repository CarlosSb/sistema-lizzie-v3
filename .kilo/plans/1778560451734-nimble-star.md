# Plano para Ajustar Página de Detalhes do Pedido

## Objetivo
Ajustar a página `frontend/src/pages/Pedidos/Detalhes.vue` para remover ações repetitivas de impressão, melhorar o fluxo de status do pedido e implementar confirmação para cancelamento de pedidos concluídos.

## Contexto Atual
A página atualmente possui um botão "Prévia" de impressão que precisa ser removido, além de ações de impressão que devem ser controladas pelo status do pedido. Também é necessário implementar uma confirmação para evitar cancelamento acidental de pedidos já concluídos.

## Regras de Negócio

### 1. Remover Botão "Prévia"
- Remover completamente o botão "Prévia" da interface
- Manter o select de impressão com as opções "Imprimir Completo" e "Imprimir Resumido"
- As opções devem continuar funcionando normalmente

### 2. Controle de Visibilidade das Ações de Impressão
- **Mostrar** ações de impressão apenas quando o pedido estiver em status final:
  - Concluído (status 4)
  - Cancelado (status 3)
- **Ocultar** ações de impressão quando o pedido estiver em status não final:
  - Aberto (status 1)
  - Pendente (status 2)
  - Ou qualquer outro status intermediário

### 3. Confirmação para Cancelamento de Pedido Concluído
- Quando tentar alterar status de "Concluído" para "Cancelado":
  - Não aplicar diretamente
  - Abrir modal de confirmação
  - Informar que a ação invalidará o pedido
  - Só aplicar após confirmação explícita

### 4. Modal de Confirmação
**Título:** "Cancelar pedido concluído?"

**Texto:**
"Este pedido já foi concluído. Ao cancelar, o pedido será invalidado e essa alteração ficará registrada. Deseja continuar?"

**Botões:**
- "Voltar" (cancela a ação)
- "Confirmar cancelamento" (aplica o cancelamento)

## Plano de Implementação

### 1. Remover Botão "Prévia"
- Localizar e remover o botão "Prévia" do template
- Verificar que o select de impressão permanece funcional

### 2. Implementar Controle de Visibilidade
- Modificar a lógica para mostrar/ocultar o select de impressão baseado no status
- Usar computed property ou v-if baseado em `pedido.status`

### 3. Implementar Modal de Confirmação
- Adicionar estado para controlar a exibição do modal
- Criar template do modal com título, texto e botões
- Modificar `updateOrderStatus` para verificar se é mudança de concluído para cancelado
- Se for, abrir modal ao invés de aplicar diretamente
- Implementar ações dos botões do modal

### 4. Testes e Validação
- Verificar que botão "Prévia" foi removido
- Testar visibilidade das ações de impressão em diferentes status
- Testar fluxo de cancelamento de pedido concluído
- Confirmar que outras funcionalidades não foram afetadas

## Critérios de Pronto
- ✅ Botão "Prévia" removido da interface
- ✅ Select de impressão preservado e funcional
- ✅ Ações de impressão aparecem apenas para pedidos concluídos/cancelados
- ✅ Pedido concluído não pode ser cancelado sem confirmação
- ✅ Modal de confirmação funciona corretamente
- ✅ Status é atualizado apenas após confirmação
- ✅ Nenhuma outra funcionalidade da página foi alterada

  - **Mitigação:** Verificar se há conflitos com modal de PDF
