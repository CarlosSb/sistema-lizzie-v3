# Plano: Automatizar Status dos Pedidos com Base no Fluxo de Impressão

## Objetivo
Implementar mudança automática de status dos pedidos baseada nas ações de impressão, criando um fluxo suave e intuitivo. O status deve refletir o estágio do pedido: ABERTO (incompleto), PENDENTE (aguardando cliente), CONCLUÍDO (finalizado internamente).

## Análise Atual
- **Criação**: Pedido criado via Form.vue, status padrão ABERTO (1).
- **Detalhes**: Em Detalhes.vue, status pode ser alterado manualmente (ABERTO, PENDENTE, CONCLUÍDO, CANCELADO).
- **Impressão**: Botões para "Imprimir Completo" (cliente) e "Imprimir Resumido" (interno) geram PDF sem alterar status.
- **Problema**: Status não muda automaticamente, exigindo intervenção manual, o que quebra o fluxo.

## Proposta de Automação
### 1. Fluxo Automático
- **Criação → ABERTO**: Status inicial ao salvar pedido (já implementado ou padrão da API).
- **Imprimir Completo → PENDENTE**: Ao gerar documento para cliente, status muda para PENDENTE (2), indicando aguardando confirmação/cliente.
- **Imprimir Resumido**: Não altera status; versão interna para registros.
- **Concluído e Cancelamento**: Manuais via seletor (CONCLUÍDO quando entregue ao cliente, CANCELADO conforme necessário).

### 2. Implementação Simples
- Modificar `printPedido` em Detalhes.vue: Apenas para modo 'complete', setar status para PENDENTE (2) após geração do PDF.
- Para 'summary', não alterar status.
- Usar `updateOrderStatus` existente para a mudança automática.
- Manter seletor manual para CONCLUÍDO (entrega) e CANCELADO.

### 3. UX Suave
- Feedback visual: Após impressão, status atualiza automaticamente na interface.
- Sem mudanças no layout: Integração invisível aos botões existentes.
- Confirmação: Usuário vê mudança imediata, reforçando o fluxo.

## Etapas de Implementação
1. **Analisar API**: Confirmar se status padrão é ABERTO (1) na criação.
2. **Modificar printPedido**: Adicionar lógica para setar status (2 para completo, 4 para resumido) e chamar updateOrderStatus.
3. **Testar Fluxo**: Criar pedido (ABERTO) → Imprimir completo (status PENDENTE) → Imprimir resumido (status permanece PENDENTE) → Manualmente setar CONCLUÍDO na entrega.
4. **Ajustes Finos**: Garantir que mudança de status não interfira em outras operações.

## Perguntas para Usuário
- A impressão resumida deve alterar algum status interno (ex: "PROCESSADO") ou permanecer sem mudança?
- Deve haver bloqueio para não imprimir completo sem itens, ou permitir?

## Riscos
- Conflito se usuário alterar status manualmente antes de imprimir.
