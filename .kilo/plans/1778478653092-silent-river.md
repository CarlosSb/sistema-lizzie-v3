# Plano de Análise e Ajuste: Geração de PDF e Visualização Frontend

## Objetivo
Analisar se o PDF está sendo criado corretamente, ajustar a visualização no frontend da nova funcionalidade de visualização, e validar se os pedidos estão usando o novo "Ver PDF" e impressão.

## Contexto
- Backend possui PdfController@gerarPedido que retorna blob
- Frontend possui PdfViewer component com PDF.js
- Modal de visualização mostra PDF com controles de navegação e zoom
- Impressão manual abre nova janela e chama print()

## Estado Atual Identificado
- **Backend**: PdfGenerator, PedidoTemplate, TCPDF/Dompdf instalados
- **Frontend**: pdfjs-dist instalado, PdfViewer component implementado
- **Integração**: showPdfPreview() chama API e mostra modal com PdfViewer

## Possíveis Problemas Identificados
1. **Eficiência**: PdfViewer recebe pdfUrl (string), faz fetch novamente (redundante)
2. **Visualização**: Modal pode não estar dimensionado corretamente para PdfViewer
3. **Geração PDF**: Possível falha silenciosa se template ou dados inválidos
4. **Compatibilidade**: PDF.js worker pode não carregar corretamente
5. **Blob Handling**: URL revogado pode causar erro se modal fechado durante load

## Método de Análise
1. Testar geração de PDF via API call
2. Verificar display do PdfViewer no modal
3. Validar funcionamento dos botões "Ver PDF" e "Imprimir"
4. Testar em diferentes browsers
5. Verificar logs de erro no console

## Plano de Implementação

### 1. Otimizar PdfViewer para usar blob diretamente
- Modificar showPdfPreview() para passar pdfBlob ao PdfViewer
- Atualizar PdfViewer para priorizar pdfBlob sobre pdfUrl

### 2. Melhorar dimensionamento do modal
- Ajustar modal para melhor acomodar PdfViewer (altura dinâmica)
- Garantir que PdfViewer tenha espaço adequado

### 3. Adicionar validação de geração de PDF
- Verificar se blob foi criado corretamente
- Adicionar tratamento de erro se PDF vazio ou inválido
- Logs de debug para diagnóstico

### 4. Melhorar feedback de loading
- Loading state durante fetch do PDF no PdfViewer
- Indicador de progresso se PDF grande

### 5. Testes funcionais
- Cenário A: Pedido com itens → Ver PDF → deve mostrar PDF completo
- Cenário B: Imprimir PDF → deve abrir nova janela e imprimir
- Cenário C: Navegação e zoom no visualizador
- Cenário D: Erro na geração → deve mostrar mensagem amigável

## Critérios de Pronto
- PDF gerado corretamente e visualizado no modal
- Controles de navegação e zoom funcionam
- Impressão abre nova janela corretamente
- Tratamento adequado de erros
- Performance aceitável (sem fetches redundantes)
